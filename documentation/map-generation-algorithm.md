# Map Generation Algorithm (Current State)

Purpose: this is the canonical map generation document for future agents.  
It replaces the old split between research notes and staged implementation planning.

## 1. Current Status

- Built-in generators: `continents`, `archipelago`.
- Architecture: contract + registry + shared deterministic pipeline.
- Output shape: `GameMap` with `tilesByKey` and `tileKeys`.
- Determinism requirement: identical `(algorithmId, width, height, seedHash, params)` must produce identical map output.

## 2. Core Architecture

## 2.1 Contracts and Registry

- `MapGenerationRequest` carries algorithm id, dimensions, seed, params, and map layout metadata.
- `MapGeneratorDefinition` defines:
  - `id`
  - optional metadata (`displayName`, `description`, `parameterDefinitions`)
  - `validateParams`
  - `generateTiles`
- `MapGeneratorRegistry` validates request, resolves generator, creates deterministic context, and builds final `GameMap`.
- Extension point for mods: `registerMapGenerator(...)` in `src/game/mapgen/index.ts`.

## 2.2 Deterministic Context

Generated by registry (`src/game/mapgen/registry.ts`):

- `random`: seeded RNG from `seedHash`.
- `createRandomStream(name)`: deterministic substream per stage.
- `noiseAt(q, r, salt?)`: deterministic hash noise.
- `noiseAtWithSeed(stream, q, r, salt?)`: deterministic stream-specific noise.
- `createRectCoords()`: stable axial coordinates for rectangular fixtures.

No `Math.random()` is allowed in generation code.

## 2.3 Pipeline Composition

Both built-in generators are thin script wrappers:

- `src/game/mapgen/generators/continents.ts`
- `src/game/mapgen/generators/archipelago.ts`

Each wrapper:

1. Validates params.
2. Builds script-specific pipeline config.
3. Calls `runGeneratorPipeline(...)`.
4. Returns generated tiles.

This keeps script behavior customizable without changing registry/contracts.

## 3. Algorithm Flow (Current)

Implemented in `src/game/mapgen/pipeline/run.ts`.

## Step 1: Grid and Subseed Setup

- Build `MapGrid` from deterministic coordinates.
- Create named subseeds (`macro`, `elevation`, `detail`, `climate`) for stage isolation.

## Step 2: Macro Structure

1. Generate Poisson seeds (`poisson.ts`).
2. Ensure minimum seed coverage for small or sparse samples.
3. Assign Voronoi region per tile (`voronoi.ts`).
4. Build macro land mask (`macro-mask.ts`), including:
   - region scoring (size, centrality, noise)
   - overshoot-aware primary region selection
   - secondary fill if map is still below macro target
   - deterministic fragmentation/growth pass
   - final target-ratio rebalance

Important anti-artifact rule: primary-region selection is intentionally kept close to target land ratio so final rebalance remains a correction, not a global reshape.

## Step 3: Tectonic Elevation

- Assign pseudo-plate vectors per region.
- Detect boundary convergence/divergence through relative plate motion.
- Produce normalized elevation + boundary intensity (`tectonics.ts`).

## Step 4: Detail Noise

- Apply rotated multi-field value-noise blending and domain warp.
- Perturb elevation and optionally mutate coastlines (`detail-noise.ts`).
- Rebalance land mask back to configured target ratio (`macro-mask.ts` helper).

## Step 5: Terrain Classification

- Convert land/water masks and elevation into layered biome + elevation output (`terrain-classify.ts`):
  - Tile terrain (biome/water): `ocean`, `deep_sea`, `coastal_sea`, `grassland`, `plains`, `desert`, `tundra`
  - Elevation: `underwater`, `flat`, `hill`, `mountain`

This separation allows combinations like plains hills or tundra mountains without encoding every combination as a terrain id.

## Step 6: Quality Metrics

- Compute map quality stats (`metrics.ts`):
  - land ratio
  - landmass count
  - largest landmass share
  - coastline complexity
  - coastline directionality score
- Additional anti-streak helper: land corridor metric (single-width linear land streak detection).

## 4. Built-In Script Parameter Models

## Continents

- `landRatio`
- `continentCountTarget`
- `tectonicStrength`
- `coastlineRoughness`
- `mountainIntensity`

Profile intent: fewer larger landmasses with stronger major-continent tendency.

## Archipelago

- `landRatio`
- `islandSizeBias`
- `chainTendency`
- `shelfWidth`
- `tectonicStrength`

Profile intent: fragmented island groups with higher ocean share and chain bias.

## 5. Debug and Repro Workflow

UI and tooling support deterministic repro capture:

- In-game debug panel can export repro payload JSON (request + metrics + digest).
- Optional full map tile data can be included.
- Replay from pasted payload is supported.
- CLI replay command:
  - `npm run mapgen:repro -- <payload.json>`

Core repro helpers:

- `src/game/mapgen/repro.ts`
- `scripts/mapgen-repro.ts`
- fixture tests in `src/game/mapgen/repro-fixtures.spec.ts`

## 6. Testing Strategy

Current automated coverage includes:

- deterministic reproducibility (same seed => same map)
- variation across seeds
- land ratio bounds
- script profile differentiation (`continents` vs `archipelago`)
- diagnostics harness aggregation
- fixture replay checks
- anti-streak corridor threshold checks on known repro seeds

All tests run with Vitest (`npm test`).
